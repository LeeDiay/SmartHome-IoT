Introduction

Base URL: http://localhost

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Endpoints

GET api/user

Example request:
curl --request GET \
    --get "http://localhost/api/user" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "http://localhost/api/user"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

Filter sensor data. This endpoint filters sensor data based on the given parameters: start date, end date, sort by attributes, and sort order.

Example request:
curl --request POST \
    "http://localhost/api/sensor-data/filter" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"start_date\": \"molestiae\",
    \"end_date\": \"quos\",
    \"sort_by\": null,
    \"sort_order\": \"officiis\"
}"

const url = new URL(
    "http://localhost/api/sensor-data/filter"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "start_date": "molestiae",
    "end_date": "quos",
    "sort_by": null,
    "sort_order": "officiis"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (200):

Request      

POST api/sensor-data/filter

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

Body Parameters

start_date   string  optional  

optional The start date for filtering data. Format: YYYY-MM-DD HH:MM:SS Example: molestiae

end_date   string  optional  

optional The end date for filtering data. Format: YYYY-MM-DD HH:MM:SS Example: quos

sort_by   string[]  optional  

optional The attributes to sort by (e.g., ['light', 'temperature']).

sort_order   string  optional  

optional The order to sort by, either 'asc' or 'desc' (default is 'asc'). Example: officiis
